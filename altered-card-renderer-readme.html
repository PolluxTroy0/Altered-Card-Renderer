Altered Card Renderer

Documentation for altered-card-renderer.js standalone canvas renderer for Altered TCG cards.
github.com/PolluxTroy0/Altered-Card-Renderer

Embed anywhere — <altered-card>

The simplest way to display cards. Drop the <script> tag and use <altered-card> wherever you want on the page — no JavaScript required.

Via jsDelivr (no hosting required):

<script src="https://cdn.jsdelivr.net/gh/PolluxTroy0/Altered-Card-Renderer@main/altered-card-renderer.js"></script>

<altered-card ref="ALT_CORE_B_AX_04_U_10"></altered-card>
<altered-card ref="ALT_EOLE_B_OR_109_U_374" locale="fr"></altered-card>

Self-hosted:

<script src="https://your-domain.com/path/to/altered-card-renderer.js"></script>

<altered-card ref="ALT_CORE_B_AX_04_U_10"></altered-card>
<altered-card ref="ALT_EOLE_B_OR_109_U_374" locale="fr"></altered-card>

Style it like any block element — set a width, drop it in a flex row, a grid, an article. The canvas inside fills the element width and maintains the card aspect ratio automatically.

Batch mode — FETCH_MODE

By default the renderer makes one API call per <altered-card> tag. Setting FETCH_MODE = 2 near the top of altered-card-renderer.js switches to batch mode: all tags on the page are collected into a single POST /api/cards/batch request, significantly reducing network round-trips when displaying several cards at once. 

// In altered-card-renderer.js — near the top:
const FETCH_MODE  = 1;   // 1 = one call per tag (default) | 2 = batch
const BATCH_SIZE  = 50;  // max references per POST request
const BATCH_MAX   = 100; // hard cap — refs beyond this are dropped

How batch mode works

When FETCH_MODE = 2, each <altered-card> element registers itself in an internal queue instead of fetching immediately. After a single browser tick (setTimeout(0)) — enough time for all tags in the page HTML to connect — the queue is flushed:

  1. Already-cached refs are served instantly from the in-memory API cache — no network request.
  2. Remaining unique refs are capped at BATCH_MAX. Refs beyond this limit are silently dropped — their elements remain empty.
  3. The uncached list is split into chunks of at most BATCH_SIZE.
  4. All chunks are sent in parallel as POST /api/cards/batch requests.
  5. Responses are merged, stored in the cache, and each waiting element is rendered with its own data.

Elements that connect after the flush (e.g. injected dynamically) automatically fall back to individual mode-1 calls, which also benefit from the shared API cache.

BATCH_SIZE and BATCH_MAX

VariableDefaultDescription
BATCH_SIZE50Maximum references per POST. If there are more unique refs, the list is split and requests are fired in parallel.
BATCH_MAX100Hard cap on the total number of refs processed. Refs beyond this are silently ignored. Set to Infinity to disable.
Example with defaults: 120 unique cards on a page → first 100 are processed (warning in console), sent as 2×50 parallel requests.

Locale in batch mode

Each <altered-card> element still respects its own locale attribute — cards on the same page can be displayed in different languages even in batch mode. The locale sent in the batch request body is taken from the first tag on the page that explicitly sets a locale attribute; if none do, it defaults to en. Because the batch API returns all locales in every response item, this choice does not affect which languages are available — it is used only for server-side logging / hints.

ModeAPI callsWhen to use

In-memory caches & hover patterns

The renderer maintains two in-memory caches shared across all card instances on the page. Both are Map objects that persist for the lifetime of the page (a full reload resets them).

Each cache can be enabled or disabled independently via a constant near the top of altered-card-renderer.js:

const CACHE_API    = true;  // card JSON from the API
const CACHE_IMAGES = true;  // frame / logo / frame-part HTMLImageElements
const CACHE_RENDER = true;  // fully rendered card as ImageBitmap
const CACHE_CANVAS = true;  // skip re-render when element already has a <canvas>
ConstantKeyWhat it stores
CACHE_APIref|localeResolved card JSON from the API. A second render of the same card never makes a network request.
CACHE_IMAGESImage URLLoaded HTMLImageElement objects — frames, set logos, frame parts, biome badges, etc. Images are never fetched twice, even across different card renders.
CACHE_RENDERref|localeImageBitmap of the fully rendered card. After the first render, subsequent renders (e.g. hover overlay) just blit the bitmap onto a new canvas — no image loading, no canvas drawing at all.
CACHE_CANVAS—Skip fetch + render entirely when connectedCallback fires on an element that already has a <canvas> (e.g. DOM move on hover).

In addition, connectedCallback skips the fetch and render entirely if the element already contains a <canvas>. This covers the case where an existing <altered-card> element is moved to an overlay or modal on hover.

Hover overlay — recommended pattern

To avoid redundant fetches and re-renders when enlarging a card on hover, prefer moving the existing element rather than cloning or recreating it: 

// On hover: move the element into an overlay — canvas guard skips re-render.
overlay.appendChild(cardEl);   // connectedCallback fires but canvas already exists → no-op

// On hover-out: move it back.
originalParent.insertBefore(cardEl, placeholder);

If your hover creates a new <altered-card> element (e.g. via innerHTML or cloneNode), the API cache still avoids the network request, but the canvas will be re-rendered from the cached data. Moving the element is always the most efficient approach.

All caches and the canvas guard are active automatically — no configuration needed.

Moving the existing element (e.g. into an overlay): canvas guard fires → zero network, zero render.
Creating a new element for the same card: render cache fires → zero network, instant bitmap blit.
First load of a card: full pipeline (API fetch → image loading → canvas draw), then all three caches are populated.

Loading placeholder

While the card images are loading, the renderer draws a placeholder on the canvas. Once loading is complete, the placeholder is always cleared — even if no background image was found (the canvas is left with the plain placeholder background, no text). A text label is drawn on top during loading — configurable via four constants near the top of altered-card-renderer.js: 

const LOADING_TEXT  = "Loading…";             // null or "" to hide
const LOADING_X     = 50;                      // horizontal position in % (0 = left, 50 = center, 100 = right)
const LOADING_Y     = 50;                      // vertical position in %   (0 = top,  50 = center, 100 = bottom)
const LOADING_COLOR = "rgba(255,255,255,0.75)"; // any CSS color

LOADING_TEXT supports two special syntaxes:

  • \n — line break. The block of lines is centred around LOADING_Y.
  • {ref} — replaced at render time by the card reference (e.g. ALT_CORE_B_AX_04_U_10).
// Examples:
const LOADING_TEXT = "Loading…\n{ref}"; // two lines: label + card ref
const LOADING_TEXT = null;              // no text at all

The font size is proportional to the canvas width (5.5%) and adapts automatically to any display size. A semi-transparent black drop shadow is always added behind the text for readability on any background.

Per-card overrides — cards_data.json

The file config/cards_data.json lets you apply forge overrides to specific cards (or groups of cards) without touching the card API data. Overrides are deep-merged into the card's forge section just before rendering — only the fields you specify are changed.

Key format

Key typeExampleMatches
Exact reference "ALT_CORE_B_AX_04_U_10" That one card only
Prefix wildcard "ALT_ALIZE_B_AX_*" Any card whose reference starts with ALT_ALIZE_B_AX_

When several patterns match the same card they are applied from least specific (shortest prefix) to most specific (exact match), so the most specific entry always wins. Keys starting with _ are treated as comments and ignored.

Example

{
  // Override bgTransform for one exact card:
  "ALT_CORE_B_AX_04_U_10": {
    "bgTransform": { "zoom": 145, "x": 55, "y": 40 }
  },

  // Override bgTransform for an entire card group:
  "ALT_ALIZE_B_AX_*": {
    "bgTransform": { "zoom": 130, "x": 50, "y": 45 }
  }
}

Built mode

When using build_renderer_for_github.py, the contents of cards_data.json are embedded into the built JS alongside the rest of the config — no separate file is needed at runtime.

External resources configuration

The renderer works in two modes depending on how it was built:

Source mode — config loaded from JSON files

The default mode. RESOURCES.embeddedConfig is null, so the renderer fetches config/index.json at startup and loads all JSON config files from configBaseUrl. The config/ folder must be present on the server.

Built mode — config embedded in the JS

Produced by build_renderer_for_github.py. The entire merged config is injected directly into the JS as embeddedConfig — no JSON files are fetched at startup. CARD_API_URL is filled from config/core.json at build time, and configBaseUrl is reset to "" so fonts and assets resolve locally. No config/ folder is needed at runtime.

Configuration variables

Declared at the top of altered-card-renderer.js: 

// Card data API URL — set at build time by build_renderer_for_github.py,
// or left empty and loaded from config/core.json at runtime (source mode).
const CARD_API_URL = "";

const RESOURCES = {
  configBaseUrl:    "https://cdn.alteredcore.org/forge/",       // forge base URL (CDN, CORS: *)
  configIndex:      "config/index.json",                       // config manifest
  alteredIconsCss:  "assets/fonts/alteredicons.css",             // Altered icon font
  qrcodeLib:        "assets/vendor/qrcodejs/qrcode.min.js",    // QR code library
  cardApiUrl:       CARD_API_URL,                                // resolved from above
  proxyUrl:         null,                                       // null | false | "https://…"
  useApiBackground: true,                                       // true = API image, false = custom URL
  backgroundUrl:    "",    // template: {ref} {locale} {faction} {rarity} {set} {id} — see cdn.alteredcore.org
  backgroundUrlIdTransform: null, // [[regexPattern, replacement], …] applied to {id} before URL substitution — null to disable
                            // e.g. [["_U_\\d+$", "_U"]] → ALT_CORE_B_AX_07_U_1698 becomes ALT_CORE_B_AX_07_U
  embeddedConfig:   null,  // set by build script — when non-null, takes full priority over configBaseUrl
};

All relative paths are resolved against configBaseUrl. Paths starting with http://, https:// or / are used as-is.

Config priority: embeddedConfig always takes precedence over configBaseUrl. In the built version, config is embedded and no JSON files are fetched at startup. To force external JSON files instead, pass embeddedConfig: null explicitly to AlteredRender.init(): 
AlteredRender.init({ embeddedConfig: null, configBaseUrl: "https://my-server.com/config/" });
To customise the API URL, edit CARD_API_URL at the top of the file — no need to touch anything else.

The forge section in the API response

The forge section is entirely optional. The renderer auto-detects faction, frame type, background image, set logo and language from the regular API fields. Only add forge fields when you need to override the defaults configured in altered-card-renderer.js:

Frame auto-selection

When forge.frameType is absent, the renderer picks the frame automatically using FRAME_AUTO_SELECT in altered-card-renderer.js. Rules are defined per rarity code and evaluated in order — the first matching rule wins. 

const FRAME_AUTO_SELECT = {
  "COMMON": [
    { frameType: "tok_c_1",     test: (d)    => _type(d) === "TOKEN" },
    { frameType: "hero_c_1",    test: (d)    => _type(d) === "HERO" },
    { frameType: "expperm_c_1", test: (d, l) => _isExpPerm(d) &&  _eff2(d) && _eff1(d, l) <  200 },
    { frameType: "expperm_c_2", test: (d, l) => _isExpPerm(d) &&  _eff2(d) && _eff1(d, l) >= 200 },
    { frameType: "expperm_c_3", test: (d, l) => _isExpPerm(d) && !_eff2(d) && _eff1(d, l) <  200 },
    { frameType: "expperm_c_4", test: (d, l) => _isExpPerm(d) && !_eff2(d) && _eff1(d, l) >= 200 },
    { frameType: "perm_c_1",    test: (d, l) => _type(d) === "PERMANENT" &&  _eff2(d) && _eff1(d, l) <  200 },
    { frameType: "perm_c_2",    test: (d, l) => _type(d) === "PERMANENT" &&  _eff2(d) && _eff1(d, l) >= 200 },
    { frameType: "perm_c_3",    test: (d, l) => _type(d) === "PERMANENT" && !_eff2(d) && _eff1(d, l) <  200 },
    { frameType: "perm_c_4",    test: (d, l) => _type(d) === "PERMANENT" && !_eff2(d) && _eff1(d, l) >= 200 },
    { frameType: "spell_c_1",   test: (d, l) => _type(d) === "SPELL" &&  _eff2(d) && _eff1(d, l) <  200 },
    { frameType: "spell_c_2",   test: (d, l) => _type(d) === "SPELL" &&  _eff2(d) && _eff1(d, l) >= 200 },
    { frameType: "spell_c_3",   test: (d, l) => _type(d) === "SPELL" && !_eff2(d) && _eff1(d, l) <  200 },
    { frameType: "spell_c_4",   test: (d, l) => _type(d) === "SPELL" && !_eff2(d) && _eff1(d, l) >= 200 },
    { frameType: "char_c_1",    test: (d, l) =>  _eff2(d) && _eff1(d, l) <  200 },
    { frameType: "char_c_2",    test: (d, l) =>  _eff2(d) && _eff1(d, l) >= 200 },
    { frameType: "char_c_3",    test: (d, l) => !_eff2(d) && _eff1(d, l) <  200 },
    { frameType: "char_c_4",    test: (d, l) => !_eff2(d) && _eff1(d, l) >= 200 },
  ],
  "RARE":    [ /* same logic, _r_ prefix — no TOKEN / HERO */ ],
  "EXALTED": [ { frameType: "char_e_1", test: () => true } ],
  "UNIQUE":  [
    { frameType: "char_u_1", test: (d, l) =>  _eff2(d) && _eff1(d, l) <  200 },
    { frameType: "char_u_2", test: (d, l) =>  _eff2(d) && _eff1(d, l) >= 200 },
    { frameType: "char_u_3", test: (d, l) => !_eff2(d) && _eff1(d, l) <  200 },
    { frameType: "char_u_4", test: (d, l) => !_eff2(d) && _eff1(d, l) >= 200 },
  ],
};
Five helper functions are available inside test():

_eff1(d, lang) — visible character count of the main effect text (API tokens and markup stripped).
_eff2(d) — true if the card has a discard / echo effect.
_type(d) — cardType.reference string, e.g. "CHARACTER", "PERMANENT", "SPELL", "TOKEN", "HERO".
_subtype(d, ref) — true if any entry in cardSubTypes[] has the given reference, e.g. "EXPEDITION".
_isExpPerm(d) — true if type=PERMANENT + subtype=EXPEDITION, or type=EXPEDITION_PERMANENT directly.

The 200-character threshold and all frame IDs are fully configurable directly in the block. If no rule matches, the renderer falls back to the defaultFrame from core.json.

Per-faction element tweaks — factionOverrides

Each entry in framedata.json → frameTypes supports an optional factionOverrides key for applying small, faction-specific adjustments to element positions or properties without creating separate frame types or touching any faction_*.json file. 

// In config/framedata.json:
"frameTypes": {
  "char_c_1": {
    "effects": { "y": 65 },
    "factionOverrides": {
      "AX": { "effects": { "y": 67, "fontSize": 16 } },
      "MU": { "cardName": { "color": "#F0E8D0" } }
    }
  }
}

Keys are faction names as returned by the API (d.faction?.name — e.g. "AX", "MU", "OR"…). Values follow the same element-property structure as the rest of the frameType entry. frameParts overrides are also supported.

The resulting defaults cascade, from lowest to highest priority:

globalDefaults[el].prop
    ↑ overridden by
frameTypes[frameType][el].prop                     // base frame type (framedata.json)
    ↑ overridden by
frameTypes[frameType].factionOverrides[faction][el].prop  // ← faction tweak (framedata.json)
    ↑ overridden by
factions[faction].types[key].defaults[el].prop     // per-frame override (faction_*.json)

Biome variant auto-detection

Each biome element (forest, mountain, ocean) has a bgVariant that determines which background badge image is displayed: zero, normal, or best. When using mountFromApi(), the variant is computed automatically from the biome values returned by the API:

  • value == 0 → "zero"
  • value == highest among the three biomes → "best" (ties: all tied biomes get "best")
  • otherwise → "normal"

The API fields to read are declared in the BIOME_VARIANT_AUTO block at the top of altered-card-renderer.js: 

const BIOME_VARIANT_AUTO = {
  forestValue:   "cardGroup.forestPower",
  mountainValue: "cardGroup.mountainPower",
  oceanValue:    "cardGroup.oceanPower",
};
// Set a value to null to disable auto-detection for that biome.

This block is independent from API_MAPPING.values — both can reference the same API fields without conflict. For forge card JSONs (mount()), the bgVariant stored in the JSON is used as-is; auto-detection only applies in mountFromApi().

QR code — API_QR_CODE

The API_QR_CODE block controls whether the QR code is shown and what URL it encodes when rendering from an API response. It sits at the top of altered-card-renderer.js alongside the other configurable blocks. 

const API_QR_CODE = {
  // true  → always show the QR code on API-rendered cards
  // false → always hide it
  // null  → use the default from config/elements.json
  visible: true,

  // URL template — {varName} placeholders are replaced by values resolved from vars below.
  // Set to null to use the value from API_MAPPING.values.qrCode instead.
  url: "https://altered.gg/{locale}/cards/{reference}",

  // Variable definitions. Each value is either:
  //   • a dot-path string  →  resolved from the API JSON
  //   • a function         →  (apiJson, lang) => string  (for computed values)
  vars: {
    reference: "reference",
    locale:    (_d, lang) => ({ en: "en-us", fr: "fr-fr", es: "es-es", it: "it-it", de: "de-de" }[lang] ?? "en-us"),
    // set: "set.reference",
  },
};

Each entry in vars is either a dot-path string (resolved from the API JSON) or a function (apiJson, lang) => string for computed values. The key is the placeholder name used in url.

If a variable resolves to null or is missing from the API JSON, its placeholder is replaced with an empty string.

Mapping format

The default mapping is defined in the API_MAPPING block at the top of altered-card-renderer.js. Each value is either a dot-path string (with optional {lang} placeholder) or a function (data, lang) => value. 

const API_MAPPING = {

  // Language — read from the forge section added by the API
  lang: "forge.lang",
  langFallback: "en",

  // Frame selection — faction from cardGroup.faction.name.
  // frameType: from forge.frameType if present, otherwise auto-selected via FRAME_AUTO_SELECT.
  // collection: from forge.collection, otherwise DEFAULT_COLLECTION.
  selection: (d, lang) => {
    const faction    = d.cardGroup?.faction?.name ?? "";
    const collection = d.forge?.collection || DEFAULT_COLLECTION;
    let   frameType  = d.forge?.frameType ?? null;
    if (!frameType) {
      const rarity = d.cardRarity?.reference ?? "";
      const match  = (FRAME_AUTO_SELECT[rarity] || []).find(r => r.test(d, lang));
      frameType    = match?.frameType ?? null;
    }
    return { faction, type: frameType ? `${collection}::${frameType}` : "" };
  },

  // Background image
  // Index into assets[] resolved from cardRarity.reference via RARITY_ASSET_INDEX.
  // Falls back to imagePath (plain string or localized object) when assets[] is absent.
  background: (d, lang) => {
    if (Array.isArray(d.assets) && d.assets.length) {
      const rarity = d.cardRarity?.reference ?? d.cardGroup?.rarity?.reference ?? "";
      const idx    = RARITY_ASSET_INDEX[rarity] ?? 0;
      return d.assets[idx] ?? d.assets[0] ?? null;
    }
    const ip = d.imagePath;
    return (ip && typeof ip === "object") ? (ip[lang] ?? ip.en ?? null) : (ip ?? null);
  },

  // Background transform
  // forge.bgTransform overrides these defaults when present in the JSON.
  bgTransform: (d) => ({
    zoom:  d.forge?.bgTransform?.zoom  ?? 120,   // default zoom — edit here to change globally
    x:     d.forge?.bgTransform?.x     ?? 50,
    y:     d.forge?.bgTransform?.y     ?? 50,
    flipX: d.forge?.bgTransform?.flipX ?? false,
  }),

  // Values — forge element id → dot-path or function
  values: {
    cardName:       (d, lang) => _loc(d.cardGroup?.name, lang),
    handCost:       "cardGroup.mainCost",
    reserveCost:    "cardGroup.recallCost",
    forestValue:    "cardGroup.forestPower",
    mountainValue:  "cardGroup.mountainPower",
    oceanValue:     "cardGroup.oceanPower",
    effects:        (d, lang) => _loc(d.cardGroup?.mainEffect, lang),
    discardEffects: (d, lang) => {            // echoEffect — array or absent
      const e = d.cardGroup?.echoEffect;
      if (!e || (Array.isArray(e) && e.length === 0)) return null;
      return Array.isArray(e) ? e.map(s => _loc(s, lang)).filter(Boolean).join("\n") : _loc(e, lang);
    },
    cardId:         "reference",
    // qrCode — handled by API_QR_CODE block (URL template + visibility).
    // Add an entry here only as fallback when API_QR_CODE.url is null.
    cardType:       (d, lang) => {
      const type = _loc(d.cardType?.name, lang) ?? "";
      const subs = (d.cardSubTypes || []).map(s => _loc(s.name, lang) ?? "").filter(Boolean).join(", ");
      return subs ? `${type} - ${subs}` : type;
    },
    artistName:     "cardGroup.illustrator",
  },
};
If a field resolves to null or undefined, the element is hidden entirely on the card. It is never replaced by a default value.

Text token substitution

Effect text from external APIs often uses shorthand tokens like {R} or []. The renderer converts these to the corresponding alteredicons font characters before drawing on canvas. The mapping is defined in the API_TEXT_TOKENS block at the top of altered-card-renderer.js:

Each entry can be a plain string (PUA character) or an object with an optional size multiplier that scales the icon relative to the surrounding text: 

const API_TEXT_TOKENS = {
  // Plain string — drawn at the default icon size
  "{R}": "\ue024",

  // Object — char + size multiplier (0.8 = 80% of current fontSize)
  "{R}": { char: "\ue024", size: 0.8 },

  // All action/keyword icons default to 0.8 — adjust per-token as needed
  "{J}": { char: "\ue026", size: 0.8 },   // Expend
  // … number icons {0}–{9}, lowercase variants…
  "[]":  " ",       // section separator → space (no size needed)
};

The size multiplier only applies to richtext elements — plain text fields (card name, type…) are unaffected. Add or remove entries to match your API's notation.

API_TEXT_TRANSFORMS — regex styling

After token substitution, the renderer applies a second pass of regex-based HTML transformations on richtext elements (effect text, discard effect, permanent text, lore text). This lets you style portions of the API text — bold keywords, coloured highlights, italic flavour text — without any changes to the API itself.

The transforms are defined in the API_TEXT_TRANSFORMS block at the top of altered-card-renderer.js: 

const API_TEXT_TRANSFORMS = [
  // #text# → coloured highlight (Altered gold)
  { pattern: /#(.*?)#/g,       replacement: '<span style="color:#C37424">$1</span>' },
  // {X} → large bold X (variable cost)
  { pattern: /\{X\}/g,         replacement: '<strong>X</strong>' },
  // (text) → italic inside parens
  { pattern: /\(([^)]+)\)/g,   replacement: '(<em>$1</em>)' },
  // [[text]] → bold + underline (keyword link)
  { pattern: /\[\[(.*?)\]\]/g, replacement: '<strong><u>$1</u></strong>' },
  // [text] → bold (keyword)
  { pattern: /\[(.*?)\]/g,     replacement: '<strong>$1</strong>' },
  // em dash → hyphen
  { pattern: /—/g,             replacement: '-' },
  // double space → line break
  { pattern: /  /g,            replacement: '\n' },
];

The resulting string is parsed as HTML — <strong>, <em>, <u>, and <span style="color:…"> are all supported by the richtext renderer. Transforms run after token substitution and only on richtext elements; plain text fields (card name, card type, etc.) are unaffected.

Missing values

If a field listed in values does not exist in the API JSON (the dot-path resolves to null or undefined), the corresponding element is hidden entirely (visible: false). It is never replaced by a config default.

For example, if artistName is mapped but the API does not return an illustrator field, the artist name element will simply not appear on the card.

Usage example

// Raw response from an external API
const apiResponse = await fetch("https://api.example.com/cards/ALT_CORE_B_AX_04_U_10")
  .then(r => r.json());

const slot = document.createElement("div");
slot.style.width = "280px";
document.body.appendChild(slot);

// Uses the API_MAPPING block from altered-card-renderer.js by default
await AlteredRender.mountFromApi(slot, apiResponse);
